# PHY RTL W1 规格流程

把协议规范变成"可交付 agent 实现、可机器判定对错"的模块级规格契约。
核心理念: W1 是**人主导的架构决策固化**, 不是"读协议自动翻译"。Claude 的角色
是协议消化的助手 + 决策陪练 + 文档固化, 而非替人拍板。

## 总体工位模型 (W1 在最前)

```
spec → [W1 规格固化] → [W2 RTL生成] → [W3 静态门] → [W4 动态验证] → [W5 回归集成]
```
W1 产出是 W2 的唯一权威输入。只有 status: frozen 的模块 spec 才允许进 W2。

## W1 两个子步骤

- **W1a 协议消化** (Claude 主力, 人 review): 从 spec 抽取某模块的行为要求、
  参数、约束、表格、图。**整理事实, 不做架构翻译**。易漏点必须主动标出
  (如暂停/恢复机制、初值拼装、边界复位), 因为 agent 直接翻译协议最易漏这些。
  注意: spec 受版权保护, 基于用户提供的内容整理, 不抄长段原文。

- **W1b 架构决策** (人主导, Claude 陪练): 协议不写、属于工程判断的部分——
  位宽、流水级、模块边界切分、接口握手、时钟域、配置空间、验证锚点。
  Claude 列权衡、提示漏点、填合理默认值并明确标注, 但决策权在人。

## 推进节奏 (三阶段, 螺旋收敛)

- **阶段A 顶层骨架**: 先做 00_toplevel.md —— 数据通路全景、各序列/模式路径、
  模块清单、全局配置空间、全局约定(时钟域等)、接口边界、开放问题清单。
  这是地基, 强耦合链路必须先有顶层再做单模块, 否则边界决策反复返工。
- **阶段B 逐模块 W1**: 套 _TEMPLATE_module.md 七章结构逐个起草。
  先用一个规格最明确的模块"打样"立模板, 再推广。
- **阶段C 收口**: 接口契约固化 (YAML→SV interface+SVA)、链路级验证计划。
W1 是迭代收敛的, 做阶段B 发现顶层漏了就回补阶段A, 这是收敛不是返工。

## 七章模块结构 (见 _TEMPLATE_module.md)

0 元信息 · 1 功能描述 · 2 接口契约 · 3 配置空间 · 4 验证意图 ·
5 架构与定点约束 · 6 完成定义 · (+ 留给人 review 的关键点)
排序逻辑: 是什么(0-1)→定边界(2-3)→怎么验(4)→怎么实现(5)→何时算完(6)。

## 字段标记约定 (必须贯穿全文)

- [协议] spec 事实 · [决策✓] 人已拍板 · [默认] Claude 填的默认值待 review · [验证] 验证意图
人一眼能分辨哪些已定、哪些待自己确认。[默认] 经 review 后改 [决策✓]。

## 判定锚点决策表 (回答"没有 golden 怎么验")

按模块类型选验收锚点, 不要给每个模块硬造 MATLAB 模型:

| 模块类型 | 主锚点 | 例 |
|---|---|---|
| 有数学参考 (调制/成型/滤波) | vs MATLAB/Python golden, 定点容差 或 EVM 上限 | SRRC, symbol_mapper |
| 控制/时序 (状态机/握手) | formal SVA 证明 | tx_ctrl_fsm |
| 可写参考模型 (编码/位操作) | vs Python behavioral model, 0容差 | fec_encoder, puncturing |
| 结构/胶水 (FIFO/CDC/总线) | formal 不变量 + protocol checker | — |
| 全新无参考 | 往返不变量 + 随机激励 (编码↔解码=原值) | — |

"没有 MATLAB 模型" ≠ "没法自动判定", 只是锚点不是数据比对。
编码/CRC/可逆操作优先用**往返不变量**, 连 golden 向量都不用造。

## 防 agent 验证不充分 (机制, 不靠自觉)

- 充分性做成 agent 绕不过的数字门槛, 写进回归脚本 exit code, 不交给 agent 判断。
- 配置组合爆炸: 约束随机 + functional/cross coverage 收敛; 离散参数用 pairwise;
  正交配置用参数化 formal。
- 第3章务必显式标"哪些维度与本模块无关(排除)", 防过度组合。
- 测试判定来源必须独立于 RTL (来自 spec/参考模型/不变量), 防测试迎合实现 bug。
- mutation kill rate 作为"测试是否真能抓 bug"的二级指标。
- 验证计划(第4章)在写测试前定义"什么算充分", 人 review 的是计划而非逐个结果。

## 交互纪律

- 关键架构决策用提问确认 (位宽/握手/边界/锚点), 一次问清影响文档的几个点, 不空转。
- 易错细节(如卷积码抽头与寄存器索引映射)必须请人亲自核对, 这是参考模型与 RTL 对齐的命门。
- 全局约束(时钟域等)定一次写进顶层, 后续模块继承, 不重复问。

## 静态门分层 (W3, 供后续)

- Verible (开源): 语法/风格 lint + format, hook 里即时跑, agent 可无限次自我修正。
- SpyGlass (商业): CDC/可综合性/latch 语义签核, 阶段性 gate, 按 license 触发。
两者互补非替代。

## 代码风格约定 (供 W2 继承)

- 中文注释, 复杂逻辑说明设计意图(位宽依据、结构选择原因)。
- 时序块所有非阻塞赋值加 #1 延迟便于波形查看 (q <= #1 d;), 含复位与正常分支。
- 兼顾 PPA, 指出时序风险; 硬件不友好的思路(除法器/超长组合逻辑)直接指出并给替代。
