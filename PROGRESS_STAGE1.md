## 5. 编排者手册(主会话每步该做什么)

1. **委派 Design**:"实现 puncturing,严格按 HANDOFF.md v__ 的接口契约" → 触发 rtl-design
2. **收 G1/G2**:读 Design 回复 + hook 的 lint 输出;语法不绿就让它继续修
3. **委派 Verify**:"为 puncturing 写验证,DUT 当黑盒,按 HANDOFF.md 接口驱动" → 触发 rtl-verify
4. **跑回归**(只有你能跑):`make <puncturing回归>`,看 exit code
5. **FAIL→过滤反馈**:按第3节规则,只给症状不给答案,委派 Design 修;轮次+1
6. **PASS→你验收 G4**:人工确认无过拟合(测试是否真覆盖规格,而非凑绿)
7. **每步更新本文件第2/3/4节**
8. **轮次到 6 仍未绿**:停,升级人工分析,记入第4节
